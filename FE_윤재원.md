# 3/25 세션 내용 정리
깃이란?
-> 변경사항 기록을 할때마다 파일이 복사되어 보관됨
-> 여러가지 부분에서 지저분한 모습이 생김 -> 깃으로 해결
깃허브는 그러한 깃들을 온라인으로 공유할 수 있게 해줌

원리
내 컴퓨터의 레퍼지토리에 커밋된 버전들을 저장
-> 깃 푸쉬를 하면 깃허브(remote)레퍼지토리에 올라감
->푸쉬 후 올라가는 원격저장소를 remote repository 라고 함

협업 시 
레포 관리자가 깃허브에서 중앙레포를 생성
중앙 레포를 fork 후 개인 레포를 생성 후 연결
이때 branch 파일을 생성하여 가지를 만듦
-> 이 branch 파일은 분기점..?이라고 생각하여 독립적인 공간이 됨
-> 작업 완료 후 해당 브랜치를 기존 코드에 병합함
-> 충돌의 최소화할 수 있다는 장점이 존재



장고 개발환경 세팅
project 와 app
app의 집합이 project
따라서 카톡의 채팅, 송금, 프로필 등의 기능들이 app이고 그 모든게 모인것을 카톡이라는 project라고 함
